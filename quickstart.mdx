# Quick Start

This guide will help you install Doc Detective and run your first test. You'll learn how to test a simple web search workflow.

<Steps>

<Step title="Install Prerequisites">

Doc Detective requires Node.js to run. Make sure you have Node.js installed:

- [Node.js](https://nodejs.org/) (tested on v20 and v22)

Verify your installation:

```bash
node --version
```

</Step>

<Step title="Install Doc Detective">

Install Doc Detective globally using npm:

```bash
npm i -g doc-detective
```

<Note>
If you don't install Doc Detective globally, you'll be prompted to install the first time you run an `npx` command.
</Note>

</Step>

<Step title="Create Your First Test">

Create a new file called `kitten-search.spec.json` with the following test:

```json kitten-search.spec.json
{
  "tests": [
    {
      "steps": [
        {
          "action": "goTo",
          "url": "https://www.duckduckgo.com"
        },
        {
          "action": "find",
          "selector": "#searchbox_input",
          "click": true,
          "typeKeys": {
            "keys": ["American Shorthair kittens", "$ENTER$"]
          }
        },
        {
          "action": "find",
          "selector": "[data-testid='web-vertical']"
        },
        {
          "action": "saveScreenshot",
          "path": "search-results.png"
        }
      ]
    }
  ]
}
```

This test:
1. Navigates to DuckDuckGo
2. Finds the search box and types "American Shorthair kittens"
3. Presses Enter to search
4. Verifies the search results appear
5. Takes a screenshot of the results

</Step>

<Step title="Run Your Test">

Run the test using the following command:

```bash
npx doc-detective --input kitten-search.spec.json
```

Doc Detective will:
- Launch a browser
- Execute each step in your test
- Display the results in your terminal
- Save a screenshot to `search-results.png`

</Step>

<Step title="View Test Results">

After the test completes, you'll see output indicating whether each step passed or failed:

```
✓ Test passed: kitten-search.spec.json
  ✓ goTo: https://www.duckduckgo.com
  ✓ find: #searchbox_input
  ✓ find: [data-testid='web-vertical']
  ✓ saveScreenshot: search-results.png
```

Check the `search-results.png` file to see the screenshot captured during the test.

</Step>

</Steps>

## Next Steps

### Try More Test Formats

Doc Detective supports multiple file formats:

<CodeGroup>

```json JSON
{
  "tests": [
    {
      "steps": [
        {
          "action": "checkLink",
          "url": "https://www.duckduckgo.com"
        }
      ]
    }
  ]
}
```

```yaml YAML
tests:
- steps:
  - action: checkLink
    url: https://www.duckduckgo.com
```

```markdown Markdown (Inline)
<!-- test -->

Go to [DuckDuckGo](https://www.duckduckgo.com) and search for **kittens**.

<!-- test end -->
```

</CodeGroup>

### Test API Documentation

Create a test for API endpoints:

```json api-test.spec.json
{
  "tests": [
    {
      "steps": [
        {
          "action": "httpRequest",
          "url": "https://reqres.in/api/users",
          "method": "post",
          "requestData": {
            "name": "morpheus",
            "job": "leader"
          },
          "responseData": {
            "name": "morpheus",
            "job": "leader"
          },
          "statusCodes": [200, 201]
        }
      ]
    }
  ]
}
```

### Test Shell Commands

Verify command-line instructions:

```json shell-test.spec.json
{
  "tests": [
    {
      "steps": [
        {
          "description": "Run a Docker container and check the output.",
          "runShell": {
            "command": "docker run hello-world",
            "stdio": "Hello from Docker!"
          }
        }
      ]
    }
  ]
}
```

### Create a Configuration File

Create a `.doc-detective.json` file to customize test behavior:

```json .doc-detective.json
{
  "runOn": [
    {
      "platforms": ["windows", "mac", "linux"],
      "browsers": [
        {
          "name": "firefox",
          "headless": false
        }
      ]
    }
  ],
  "fileTypes": [
    {
      "name": "markdown",
      "extensions": ["md", "markdown", "mdx"]
    }
  ]
}
```

Then run tests with:

```bash
npx doc-detective --config .doc-detective.json
```

<Tip>
You can override config options with command-line arguments. For example:
```bash
npx doc-detective --config .doc-detective.json --input tests.spec.json
```
</Tip>
