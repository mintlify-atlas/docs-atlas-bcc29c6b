# Doc Detective: The Documentation Testing Framework

Doc Detective is a doc content testing framework that makes it easy to keep your docs accurate and up-to-date. You write tests, and Doc Detective runs them directly against your product to make sure your docs match your user experience. Whether it's a UI-based process or a series of API calls, Doc Detective can help you find doc bugs before your users do.

## Why Doc Detective?

Doc Detective ingests test specifications and text files, parses them for testable actions, then executes those actions in a browser. The results (PASS/FAIL and context) are output as a JSON object so that other pieces of infrastructure can parse and manipulate them as needed.

## Key Features

### Test UI Workflows

Test user interface interactions like clicks, navigation, and form inputs to ensure your documentation matches the actual user experience.

### Validate API Calls

Verify API documentation by making real HTTP requests and validating responses against expected values.

### Run Shell Commands

Test command-line instructions by executing shell commands and checking their output.

### Multiple File Formats

Write tests in JSON, YAML, or embed them directly in your Markdown documentation with inline test syntax.

### Cross-Platform Support

Run tests across Windows, Mac, and Linux with support for multiple browsers including Chrome, Firefox, and Safari.

### Flexible Integration

Integrate Doc Detective into your CI/CD pipeline, documentation build process, or run tests on-demand.

## What You Can Test

Doc Detective supports a wide range of actions to test your documentation:

- **checkLink**: Verify URLs return acceptable status codes
- **click**: Click elements with specified text or CSS selectors
- **find**: Check if elements exist and optionally interact with them
- **goTo**: Navigate to specific URLs
- **httpRequest**: Make HTTP requests to test APIs
- **runShell**: Execute shell commands
- **runCode**: Run code snippets
- **type**: Type text and special keys
- **wait**: Add pauses between actions
- **screenshot**: Capture screenshots for visual verification
- **record**: Record video of test execution

## Components

Doc Detective has multiple components to integrate with your workflows:

- **Doc Detective**: A standalone tool that enables testing without a separate Node project
- **Doc Detective Core**: An NPM package that provides the testing functionality
- **Doc Detective Docs**: Documentation and guides at [doc-detective.com](https://doc-detective.com)
